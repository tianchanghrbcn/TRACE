# TRACE Data Directory

This directory stores data needed by the TRACE artifact.

## Layout

- `data/raw/train/`: public benchmark data used by the cleaning-clustering pipeline.
- `data/pre_experiment/`: small pre-experiment inputs used to regenerate alpha/weight figures.
- `data/external/`: reserved for future downloaded or externally supplied data.
- `data/processed/`: reserved for local processed data. Generated outputs should not be committed.

## Publication policy

The benchmark data required for reviewer-facing reproducibility should be available in the artifact when redistribution is allowed by the original data source.

Generated outputs, repaired data, clustered results, logs, and figures are not stored as source artifacts by default. They are regenerated by scripts.

If a future dataset has unclear redistribution rights, the dataset should not be committed directly. Instead, provide source metadata, download instructions, or a synthetic replacement for smoke testing.
