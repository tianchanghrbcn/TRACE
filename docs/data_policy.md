# Data Policy

TRACE uses benchmark tabular datasets to evaluate how data cleaning affects unsupervised clustering.

## What is public in the repository

The reviewer-facing artifact keeps the data needed for reproducibility under:

- `data/raw/train/`
- `data/pre_experiment/`

These files are intended to support Mode B smoke validation and Mode A result replay.

## What is not committed by default

The following are generated artifacts and should not be committed as source files:

- `results/processed/*.csv`
- `results/tables/*.csv`
- `results/logs/*`
- `results/cleaned_data/*`
- `results/clustered_data/*`
- `figures/**/*.png`
- `figures/**/*.pdf`
- generated pre-experiment outputs
- generated visual-demo outputs

They can be regenerated by documented scripts.

## Dataset redistribution rule

A dataset may be committed only when redistribution is allowed by its source license or when it is a synthetic/demo dataset created for TRACE.

If a dataset has unclear redistribution rights, use one of these alternatives:

1. provide a download script and source URL;
2. provide a metadata record and checksum;
3. provide a synthetic replacement for smoke testing;
4. keep the dataset out of the public release.

## Current artifact decision

For advisor-review v0.1.1, the required reproducibility data are kept under `data/raw/train/` and `data/pre_experiment/`. Generated artifacts remain ignored and reproducible.
