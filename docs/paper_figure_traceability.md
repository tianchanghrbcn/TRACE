# Paper Figure Traceability

Stage 3R.5.3 validates whether paper figures are traceable to archived and generated outputs.

## Run

First run the figure harness:

    python scripts/57_select_paper_figure_sources.py
    python scripts/58_run_paper_figure_scripts.py --clean --timeout 1200
    python scripts/59_validate_paper_figure_outputs.py

Then run traceability validation:

    python scripts/60_validate_paper_figure_traceability.py

## Outputs

Generated reports:

- `analysis/paper_generated/paper_figures/paper_figure_traceability_report.json`
- `analysis/paper_generated/paper_figures/paper_figure_traceability_report.md`

## Validation meaning

- `PASS_ARCHIVED_AND_GENERATED`: the paper figure reference has both archived and generated traceability.
- `WARN_ARCHIVED_ONLY`: the figure exists in archived paper outputs but is not regenerated by the current figure harness.
- `WARN_NO_ARCHIVED_REFERENCE`: generated output exists, but no archived reference with the same figure name was found.
- `FAIL_NO_REFERENCE`: neither archived nor generated evidence was found for a LaTeX figure reference.

## Scope

This step validates traceability, not pixel-identical visual equivalence.

For figures, byte-level equality is not required because PDF metadata, fonts, and rendering environments may differ.
